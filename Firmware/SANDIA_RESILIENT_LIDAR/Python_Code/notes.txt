PC side receives a frame header = {0xAA, 0x55, 0xAA, 0x55}
That tells the PC that a frame transfer is about to begin
    - eventually, there will also need to be other types of communication

Row order of the pixels needs to be reconstructed
    - epc transmits in this order:
       Send row 120, 119, 121, 118, 122, 117, 123 etc. all the way to 240, 1

If the data received is 0x0FFF, then that means the pixel is saturated and shouldn't be used for depth
